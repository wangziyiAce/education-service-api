  1. 原 pre-check 竞争问题的修复方式

  删除了 tools.py 中的非原子 pre-check：

  # ❌ 旧代码（Iteration 1.2）
  created = True
  if idempotency_key:
      existing = db.query(ReportGeneration).filter_by(
          idempotency_key=idempotency_key
      ).first()
      if existing is not None:
          created = False

  report = create_report_task(...)

  两个请求可能同时查询到 None，都得到 created=True，即使其中一个最终因 UNIQUE INDEX 冲突而失败。

  替换为：直接使用 create_report_task_result()，由 orchestrator 在数据库事务内原子判断 created。

  2. created 标志现在如何产生

  由 create_report_task_result() 在 orchestrator 中根据真实的数据库操作结果产生，三条路径：

  ┌────────┬──────────────────────────────────────────────────────┬─────────┐
  │  路径  │                         条件                         │ created │
  ├────────┼──────────────────────────────────────────────────────┼─────────┤
  │ 路径 1 │ 提前命中已有 idempotency_key                         │  False  │
  ├────────┼──────────────────────────────────────────────────────┼─────────┤
  │ 路径 2 │ 当前事务 INSERT + commit 成功                        │  True   │
  ├────────┼──────────────────────────────────────────────────────┼─────────┤
  │ 路径 3 │ commit 触发 IntegrityError → rollback → 查询胜出记录 │  False  │
  └────────┴──────────────────────────────────────────────────────┴─────────┘

  3. UNIQUE 冲突后 created 的值

  False。IntegrityError 被捕获 → db.rollback() → 按 idempotency_key 查询胜出记录 → 返回 ReportTaskCreationResult(report=winner, created=False)。

  9. generate_report() 是否原子领取

  是。从 Iteration 1.3 起使用条件 UPDATE：

  updated_rows = (
      db.query(ReportGeneration)
      .filter(
          ReportGeneration.id == report_id,
          ReportGeneration.status == "pending",
      )
      .update({"status": "generating", "started_time": now})
  )
  db.commit()

  if updated_rows != 1:
      db.refresh(report)
      return report  # 已被其他 worker 领取

  10. 原子领取的具体实现

  使用 SQLAlchemy 条件 UPDATE（WHERE status == 'pending'），数据库在行级保证只有一个 worker 能将 pending 切换为 generating。updated_rows == 0
  时直接返回不执行生成逻辑。

  测试验证：
  - generating → updated_rows=0（不能被领取）
  - completed → updated_rows=0
  - failed → updated_rows=0
