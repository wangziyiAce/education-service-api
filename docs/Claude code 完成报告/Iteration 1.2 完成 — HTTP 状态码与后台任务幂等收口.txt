  1. HTTP 状态码修复方式

  在 routers/report_assistant.py 中注入 fastapi.Response 对象，根据 result.status 动态设置 response.status_code，而非在 @router.post 装饰器上使用固定的 status_code。

  def handle_assistant_message(
      request: ReportAssistantMessageRequest,
      response: Response,  # 新增
      background_tasks: BackgroundTasks,
      ...
  ):
      result = service.handle_message(...)

      if result.status == "generating":
          response.status_code = status.HTTP_202_ACCEPTED
      elif result.status == "permission_denied":
          response.status_code = status.HTTP_403_FORBIDDEN
      else:
          response.status_code = status.HTTP_200_OK

      return result

  11. 已知竞争条件

  pre-check 与 create_report_task 之间的窗口：

  tools.py: existing = db.query(...).first()   ← 线程 A: found None
  tools.py: existing = db.query(...).first()   ← 线程 B: found None (A 还没 commit)
  tools.py: create_report_task(...)             ← 线程 A: db.add + db.commit ✅
  tools.py: create_report_task(...)             ← 线程 B: UNIQUE INDEX 冲突 → 返回已有记录

  两个线程都通过 pre-check（created=True），都会注册后台任务。但：
  - 数据层安全：DB UNIQUE INDEX 保证只有一条记录
  - 调度层安全：generate_report_async 内部 generate_report() 先检查 status（pending→generating），第二次调用时状态已是 generating，不会重复执行
