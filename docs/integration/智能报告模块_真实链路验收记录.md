# 智能报告模块真实链路验收记录

## 验收范围

本记录对应阶段 2，只验证开发环境中的真实 MySQL、真实 DeepSeek、报告异步生成、助手追问、Evidence 与周期比较。未执行 Docker、预发布或生产上线。

## 环境结论

- MySQL：8.0.46，目标库 `education_service` 可连接。
- 报告核心表：8/8 存在。
- LLM：`deepseek-v4-pro`，通过项目 `ReportLLMClient` 真实调用。
- 配置：`REPORT_AI_MODE=llm`；密钥仅存在于被 Git 忽略的本地 `.env`。
- 安全：本记录、代码和测试不保存密钥、Prompt 原文或模型完整回答。

## 本阶段修复

1. 补齐 LLM Intent Prompt 中遗漏的 Iteration 3 意图和 `comparison_relative_period`。
2. 当模型识别 `compare_reports` 但漏填报告类型时，由 Python 从受控报告目录关键词中补齐唯一类型。
3. 把比较工具已经计算出的周期、指标和质量门禁显式写入公开响应，避免前端收到空比较面板。

以上补全均受报告 Registry、关键词目录、Pydantic Schema 和权限检查约束，不允许模型自造报告类型、指标或数值。

## 数据准备说明

确定性比较解析器在周日仍选择最近两个已经结束的完整自然周。数据库原来缺少 2026-06-22 至 2026-06-28 的基线报告，因此通过正式生成接口创建报告 10；它与 2026-06-29 至 2026-07-05 的已有报告组成可比较周期。没有手工伪造响应或直接修改报告内容。

## 已知限制

- 登录接口使用统一 `{code,data,message}` 封装，部分报告接口返回裸 Schema；统一响应体验留到前端集成阶段处理。
- 比较回答的 LLM 自然语言生成若未通过证据保护，会按既有规则降级为确定性模板；结构化 MetricComparison、Evidence ID 和 DataQuality 仍由 Python 返回。
- `.pytest_cache` 当前存在 Windows 权限警告，不影响测试执行结果。
