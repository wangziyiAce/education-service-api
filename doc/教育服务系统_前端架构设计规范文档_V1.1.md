## 1. 设计总则

### 1.1 前端定位

```text
┌─────────────────────────────────────────────────────────────┐
│  接入层（Presentation Layer）                                 │
│                                                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │  Dify WebApp │  │  Vue 3 后台  │  │   Swagger    │       │
│  │  (AI 对话)   │  │  (管理操作)   │  │  (接口测试)  │       │
│  │  P0 演示     │  │  P1 开发     │  │  P0 必备     │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
│                         │                                    │
│                    所有请求通过 FastAPI                       │
│                    不直接操作 MySQL                           │
└─────────────────────────┬───────────────────────────────────┘
                          │
              ┌───────────▼───────────┐
              │  FastAPI 业务服务层    │
              └───────────────────────┘
```

### 1.2 前端建设策略（对齐架构文档 8.1 节）

| 方案 | 作用 | 优先级 | 适用场景 |
|------|------|--------|---------|
| **Dify WebApp** | 快速演示 AI 对话能力 | P0 | 客服对话、意图识别、RAG 问答 |
| **Swagger** | 快速演示接口能力 | P0 | 接口测试、答辩演示 |
| **Vue 3 + Element Plus** | 展示后台管理页面 | P1 | CRM 管理、请假审批、报告查看 |

> **策略**：时间紧张时优先保证 Dify WebApp + Swagger 演示，Vue 后台按页面优先级分批完成。

### 1.3 设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **字段名与 API 响应对齐** | 前端字段名对应 API 返回的 JSON 字段名（均对齐数据库字段名） |
| 2 | **不直接操作数据库** | 所有数据通过 FastAPI 接口获取 |
| 3 | **组件化开发** | 复用通用组件（表格、表单、状态标签、分页等） |
| 4 | **乐观更新 + 错误回滚** | 状态变更先更新 UI，失败时回滚 |
| 5 | **权限控制前端兜底** | 路由守卫 + 按钮级权限 + API 返回数据权限 |
| 6 | **错误边界** | 每个页面/组件独立处理错误，避免整个应用崩溃 |
| 7 | **响应式设计** | 支持 1920px 和 1366px 两种常见分辨率 |
| 8 | **Token 管理** | MVP 使用 localStorage，后续升级 HttpOnly Cookie |

### 1.4 全链路评审定稿结论（V1.1）

前端 V1.1 在 V1.0 基础上收敛 Vue 后台范围，避免把 3 天 MVP 的主链路压到页面开发上。经全链路评审，最终口径如下：

| 评审项 | 定稿结论 |
|---|---|
| P0 演示入口 | Dify WebApp 展示 AI 对话能力，Swagger 展示 FastAPI 接口能力 |
| Vue 优先级 | Vue 3 后台属于 P1 增强，不阻塞 P0 主链路验收 |
| 字段映射 | TypeScript 类型字段继续与 API V1.2 / 数据库 V2.1 使用同名 `snake_case` |
| 权限边界 | 前端只做路由和按钮兜底，真实权限与数据隔离由 FastAPI 完成 |
| 客户研判 | 前端上传页作为 P1/P2 页面，文本研判链路优先，复杂文件上传后续增强 |
| 定稿状态 | 本文档可作为 Vue 项目结构、组件划分、API 封装和页面排期依据 |

### 2.2 架构分层

```text
┌──────────────────────────────────────────────────────┐
│  视图层 (Views)                                       │
│  - 登录页、首页看板、CRM客户页、学生服务页、报告中心    │
│  - 每个 View 对应一个路由                              │
├──────────────────────────────────────────────────────┤
│  组件层 (Components)                                  │
│  - 通用组件：StatusTag、DataTable、SearchForm、...     │
│  - 业务组件：LeadCard、EventCard、ReportCard、...      │
│  - 布局组件：AppLayout、Sidebar、Header、...           │
├──────────────────────────────────────────────────────┤
│  逻辑层 (Composables / Hooks)                         │
│  - useAuth、usePagination、usePermission、...          │
│  - useCrud（通用 CRUD 逻辑封装）                        │
├──────────────────────────────────────────────────────┤
│  数据层 (Stores / API)                                │
│  - Pinia Stores：authStore、crmStore、studentStore    │
│  - API 模块：authApi、crmApi、eventApi、reportApi     │
│  - 请求拦截器：Token 注入、错误统一处理、响应格式化      │
├──────────────────────────────────────────────────────┤
│  基础层 (Utils / Constants / Types)                   │
│  - 工具函数：日期格式化、状态映射、数据脱敏              │
│  - 常量定义：状态枚举、错误码映射、路由路径              │
│  - TypeScript 类型：API 请求/响应类型、业务实体类型      │
└──────────────────────────────────────────────────────┘
```

### 2.3 组件分层规则

```text
Views  →  页面级组件，负责组装子组件、调用 Store
  │
Components  →  可复用组件，通过 props/emits 通信
  │
Composables  →  逻辑封装，不渲染 DOM
  │
Stores  →  全局状态，跨组件共享
  │
API  →  数据请求，与后端通信
```

**禁止规则：**
1. View 不直接操作 DOM
2. Component 不直接调用 API（通过 Store 或 Composable）
3. Store 不包含 UI 逻辑
4. Composables 不持有全局状态（状态放 Store）

## 5. 组件设计规范

### 5.1 通用组件设计

#### StatusTag — 状态标签组件

```vue
<!-- components/common/StatusTag.vue -->
<template>
  <el-tag :type="tagType" :size="size" :effect="effect">
    {{ label }}
  </el-tag>
</template>

<script setup lang="ts">
import { computed } from 'vue'

const props = withDefaults(defineProps<{
  status: string
  size?: 'small' | 'default' | 'large'
  effect?: 'light' | 'dark' | 'plain'
  category?: 'lead' | 'event' | 'leave' | 'feedback' | 'psych' | 'report'
}>(), {
  size: 'small',
  effect: 'light',
  category: 'lead'
})

// ⭐ 状态映射表（对齐数据库 ENUM 值）
const STATUS_MAP: Record<string, Record<string, { label: string; type: string }>> = {
```

## 6. API 对接层设计

### 6.1 Axios 实例配置

```typescript
// api/request.ts
import axios from 'axios'
import type { AxiosInstance, AxiosRequestConfig, AxiosResponse } from 'axios'
import { ElMessage } from 'element-plus'
import router from '@/router'

// ⭐ 统一响应格式（对齐 API 文档 2.3 节）
interface ApiResponse<T = any> {
  code: number
  message: string
  data: T
}

const instance: AxiosInstance = axios.create({
  baseURL: '/api/v1',           // 通过 Vite 代理转发
  timeout: 15000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 请求拦截器：注入 Token
instance.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('access_token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器：统一错误处理
instance.interceptors.response.use(
  (response: AxiosResponse<ApiResponse>) => {
    const { code, message, data } = response.data
    
    if (code === 0) {
      return data  // 直接返回 data，简化业务层处理
    }
    
    // 业务错误码处理
    handleBusinessError(code, message)
    return Promise.reject({ code, message })
  },
  (error) => {
    // HTTP 错误处理
    if (error.response) {
      const { status } = error.response
      switch (status) {
        case 401:
          localStorage.removeItem('access_token')
          router.push('/login')
          ElMessage.error('登录已过期，请重新登录')
          break
        case 403:
          ElMessage.error('无权限访问')
          break
        case 404:
          ElMessage.error('请求的资源不存在')
          break
        case 500:
          ElMessage.error('服务器内部错误')
          break
        default:
          ElMessage.error(`请求失败 (${status})`)
      }
    } else if (error.code === 'ECONNABORTED') {
      ElMessage.error('请求超时，请稍后再试')
    } else {
      ElMessage.error('网络异常，请检查网络连接')
    }
    return Promise.reject(error)
  }
)

// ⭐ 业务错误码处理（对齐 API 文档第 3 章）
function handleBusinessError(code: number, message: string) {
  switch (code) {
    case 40001:
      ElMessage.warning(message || '参数错误')
      break
    case 40101:
    case 40102:
    case 40103:
      localStorage.removeItem('access_token')
      router.push('/login')
      ElMessage.error(message || '认证失败')
      break
    case 40401:
    case 40402:
      ElMessage.warning(message || '资源不存在')
      break
    case 40901:
    case 40902:
      ElMessage.warning(message || '操作冲突')
      break
    case 42201:
    case 42202:
    case 42203:
    case 42204:
    case 42205:
      ElMessage.warning(message || '业务规则校验失败')
      break
    case 50201:
    case 50202:
      ElMessage.error(message || 'AI 服务异常')
      break
    default:
      ElMessage.error(message || '操作失败')
  }
}

export default instance
```

### 7.1 Pinia Store 设计原则

```text
Store 职责：
  - 管理全局共享状态（用户信息、权限、当前选中的客户等）
  - 缓存 API 请求结果（减少重复请求）

  - 不直接操作 DOM
  - 不在 Store 之间循环引用
```

```typescript
// stores/auth.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { login as loginApi, getCurrentUser } from '@/api/auth'
import type { User } from '@/types/models'

export const useAuthStore = defineStore('auth', () => {
  const user = ref<User | null>(null)
  const token = ref<string>(localStorage.getItem('access_token') || '')

  const isLoggedIn = computed(() => !!token.value && !!user.value)
  const userType = computed(() => user.value?.user_type)           // ⭐ 对齐 sys_user.user_type
  const realName = computed(() => user.value?.real_name)           // ⭐ 对齐 sys_user.real_name

  async function login(username: string, password: string) {
    const result = await loginApi({ username, password })
    token.value = result.access_token
    user.value = {
      user_id: result.user_id,
      username: result.username,
      real_name: result.real_name,
      user_type: result.user_type,
      role_id: result.role_id,
    }
    localStorage.setItem('access_token', result.access_token)
    return result
  }

  function logout() {
    token.value = ''
    user.value = null
    localStorage.removeItem('access_token')
  }

  function hasAnyRole(roles: string[]): boolean {
    if (!user.value) return false
    return roles.includes(user.value.user_type)
  }

  async function fetchCurrentUser() {
    if (!token.value) return
    const u = await getCurrentUser()
    user.value = {
      user_id: u.user_id,
      username: u.username,
      real_name: u.real_name,
      user_type: u.user_type,
      role_id: u.role_id,
      department: u.department,
    }
  }

  return { user, token, isLoggedIn, userType, realName, login, logout, hasAnyRole, fetchCurrentUser }
})
```

```typescript
import { defineStore } from 'pinia'

import { getLeads, getLeadDetail, createLead, updateLeadStatus, createFollowUp } from '@/api/crm'

    }

    if (index !== -1) {
      leads.value[index].status = status
    }
    return result
  }

    return await createFollowUp(leadId, data)
  }
```

### 7.4 异步任务轮询 Store（报告生成）

```typescript
// composables/usePolling.ts
import { ref, onUnmounted } from 'vue'

export function usePolling<T>(
  fetchFn: () => Promise<T>,
  checkFn: (data: T) => boolean,  // 返回 true 表示完成
  interval = 2000,
  maxAttempts = 60
) {
  const data = ref<T | null>(null)
  const isPolling = ref(false)
  const attempts = ref(0)
  let timer: ReturnType<typeof setInterval> | null = null

  function start() {
    if (isPolling.value) return
    isPolling.value = true
    attempts.value = 0
    poll()
  }

  async function poll() {
    timer = setInterval(async () => {
      if (attempts.value >= maxAttempts) {
        stop()
        return
      }
      attempts.value++
      try {
        data.value = await fetchFn()
        if (checkFn(data.value!)) {
          stop()
        }
      } catch {
        stop()
      }
    }, interval)
  }

  function stop() {
    if (timer) {
      clearInterval(timer)
      timer = null
    }
    isPolling.value = false
  }

  onUnmounted(stop)

  return { data, isPolling, attempts, start, stop }
}
```

```vue
<script setup lang="ts">
import { ref, computed, onMounted, reactive } from 'vue'
import { useRoute, useRouter } from 'vue-router'
import { ElMessage } from 'element-plus'
import { useCrmStore } from '@/stores/crm'
import { getFollowUps } from '@/api/crm'
import PageHeader from '@/components/common/PageHeader.vue'
import StatusTag from '@/components/common/StatusTag.vue'
import FollowUpTimeline from '@/components/business/FollowUpTimeline.vue'
import { formatPhone } from '@/utils/desensitize'
import { formatTime } from '@/utils/format'
import type { Lead, FollowUp } from '@/types/models'

const route = useRoute()
const router = useRouter()
const crmStore = useCrmStore()

const lead = ref<Lead | null>(null)
const followUps = ref<FollowUp[]>([])
const loading = ref(false)

const genderMap: Record<string, string> = { M: '男', F: '女', U: '未知' }

// ⭐ 状态机约束（对齐 API 文档 6.4 节 + 数据库文档 6.4.1 节）
const STATUS_TRANSITIONS: Record<string, string[]> = {
  new:         ['contacting', 'lost'],
  contacting:  ['qualified', 'lost'],
  qualified:   ['signed', 'lost'],
  signed:      [],
  lost:        [],
}

const canChangeStatus = computed(() => {
  if (!lead.value) return false
  return STATUS_TRANSITIONS[lead.value.status]?.length > 0
})

const availableStatuses = computed(() => {
  if (!lead.value) return []
  const transitions = STATUS_TRANSITIONS[lead.value.status] || []
  const statusLabels: Record<string, string> = {
    contacting: '跟进中',
    qualified: '已确认',
    signed: '已签约',
    lost: '已流失',
  }
  return transitions.map(v => ({ value: v, label: statusLabels[v] }))
})

const newStatus = ref('')
const lostReason = ref('')

const followForm = reactive({
  follow_type: 'phone',
  content: '',
  next_plan: ''
})

onMounted(async () => {
  loading.value = true
  const leadId = Number(route.params.id)
  lead.value = await crmStore.fetchLeadDetail(leadId)
  const result = await getFollowUps(leadId)
  followUps.value = result.items
  loading.value = false
})

async function handleStatusUpdate() {
  if (!lead.value) return
  if (newStatus.value === 'lost' && !lostReason.value) {
    ElMessage.warning('变更为流失状态必须填写流失原因')
    return
  }
  try {
    await crmStore.changeLeadStatus(lead.value.id, newStatus.value, lostReason.value)
    lead.value.status = newStatus.value
    ElMessage.success('状态更新成功')
    newStatus.value = ''
    lostReason.value = ''
  } catch {
    // 错误已在拦截器处理
  }
}

async function handleAddFollowUp() {
  if (!lead.value || !followForm.content) return
  try {
    await crmStore.addFollowUp(lead.value.id, {
      follow_type: followForm.follow_type,
      content: followForm.content,
      next_plan: followForm.next_plan,
    })
    ElMessage.success('跟进记录已添加')
    followForm.content = ''
    followForm.next_plan = ''
    // 刷新跟进列表
    const result = await getFollowUps(lead.value.id)
    followUps.value = result.items
  } catch {
    // 错误已在拦截器处理
  }
}
</script>
```

### 11.3 数据级权限（前端兜底）

> **注意**：数据权限主校验在 FastAPI Service 层完成（对齐 API 文档 13.4 节），前端只做 UI 层面的兜底展示。

```typescript
// composables/usePermission.ts
export function usePermission() {
  const authStore = useAuthStore()

  // 是否可以看到客户数据
  function canViewLead(lead: Lead): boolean {
    if (authStore.hasAnyRole(['admin'])) return true
    if (authStore.hasAnyRole(['manager'])) {
      // 经理看本部门（前端通过 department 字段判断）
      return lead.owner_department === authStore.user?.department
    }
    // 员工只能看自己的
    return lead.owner_employee_id === authStore.user?.user_id
  }

  // 是否可以操作请假审批
  function canApproveLeave(leave: LeaveRequest): boolean {
    if (authStore.hasAnyRole(['admin', 'team_leader'])) return true
    return false
  }

  // 是否可以查看心理数据
  function canViewPsychData(studentId: number): boolean {
    if (authStore.hasAnyRole(['admin'])) return true
    if (authStore.hasAnyRole(['team_leader'])) {
      // 班主任只能看自己负责的学生（通过 student_info.class_teacher_id）
      // 前端无法直接判断，依赖 API 返回的数据权限
      return true  // API 层会做最终过滤
    }
    return authStore.user?.user_id === studentId
  }

  return { canViewLead, canApproveLeave, canViewPsychData }
}
```

### 11.4 数据隔离矩阵（前端视角）

| 数据类型 | 前端可见角色 | 前端判断方式 |
|---------|------------|------------|
| CRM 客户列表 | 员工看自己的，经理看部门，管理员看全部 | API 已过滤 + `canViewLead` 兜底 |
| 请假记录 | 学生看自己的，班主任看负责的，管理员看全部 | API 已过滤 |
| 投诉工单 | 学生看自己的，处理人看指派的，管理员看全部 | API 已过滤 |
| 心理预警 | 仅班主任和管理员 | 路由级权限 + API 已过滤 |
| 心理记录 | 仅授权老师和管理员 | 路由级权限 + API 已过滤 |
| 报告 | 所有人（报告不含敏感数据） | 登录即可 |
